# “链踪” 企业级部署指南 (V8.x)

## 引言

欢迎阅读“链踪”的企业级部署指南。本指南旨在为您提供一个清晰、分步的流程，将“链踪”应用程序成功地部署到您公司的生产服务器环境中，使其成为一个稳定、安全、可供网络团队使用的协作平台。

整个部署过程分为两个核心部分：
1.  **后端代理 (Backend Agent)**: 运行在公司服务器上的Python应用，是整个系统的“大脑”。
2.  **前端应用 (Frontend Application)**: 用户通过浏览器访问的React Web应用，是系统的“脸面”。

---

## 2. 后端代理部署 (Backend Agent)

这是部署的核心部分，请在一台专用服务器上执行（推荐使用Linux，如Ubuntu Server 22.04 LTS）。

### **2.1. 准备工作**
1.  **准备服务器**: 确保您有一台已安装 **Python 3.8 或更高版本** 的服务器。
2.  **拷贝文件**: 将项目中的所有后端相关文件，拷贝到服务器的一个专用目录中 (例如: `/opt/chaintrace-agent/`)。
    *   `agent.txt` -> `agent.py`
    *   `core.txt` -> `core.py`
    *   `services.txt` -> `services.py`
    *   `api_routes.txt` -> `api_routes.py`
    *   `websocket_handler.txt` -> `websocket_handler.py`
    *   `database.txt` -> `database.py`
    *   `models.txt` -> `models.py`
    *   `crud.txt` -> `crud.py`
    *   `config.txt` -> `config.ini`
    *   `requirements.txt`
3.  **创建虚拟环境 (推荐)**: 为隔离项目依赖，强烈建议创建Python虚拟环境。
    ```bash
    cd /opt/chaintrace-agent/
    python3 -m venv venv
    source venv/bin/activate
    ```

### **2.2. 安装与配置**
1.  **安装依赖**:
    ```bash
    pip install -r requirements.txt
    ```
2.  **配置 `config.ini`**: 这是**最关键**的一步。请使用文本编辑器打开 `config.ini` 文件，并根据您的生产环境进行配置。
    *   **`[server]`**:
        *   `host`: **必须**设置为 `0.0.0.0`，以允许服务器监听所有网络接口，从而接受来自局域网的连接。
        *   `port`: 可以使用默认的 `8001`，或根据需要更改为其他未被占用的端口。
    *   **`[gemini]`**: 填入公司申请的、用于生产环境的 **Google Gemini API 密钥**。
    *   **`[credentials_*]`**: 定义所有需要用到的设备登录凭证。为安全起见，建议使用权限受限的、专门用于自动化的服务账户。
    *   **`[interception_rules]`**: 根据公司的安全策略，配置需要实时拦截的命令。
    *   **`[device_map]`**: 详细、准确地填入所有需要管理的设备的真实IP、Netmiko设备类型和对应的凭证区段名称。

### **2.3. 运行与持久化**
1.  **首次运行**: 在激活虚拟环境后，执行 `python agent.py`。这将完成数据库 `chaintrace.db` 的初始化。检查日志确保无误后，按 `Ctrl+C` 停止。
2.  **生产环境运行 (使用 systemd)**: 为了让代理服务能在后台持续运行并在服务器重启后自动启动，我们使用 `systemd`。
    *   **创建服务文件**:
        ```bash
        sudo nano /etc/systemd/system/chaintrace.service
        ```
    *   **粘贴以下内容** (请根据您的实际路径修改 `WorkingDirectory` 和 `ExecStart`):
        ```ini
        [Unit]
        Description=ChainTrace Backend Agent
        After=network.target

        [Service]
        User=your_user          # 推荐使用一个非root的专用用户
        Group=your_group
        WorkingDirectory=/opt/chaintrace-agent
        ExecStart=/opt/chaintrace-agent/venv/bin/uvicorn agent:app --host 0.0.0.0 --port 8001
        Restart=always

        [Install]
        WantedBy=multi-user.target
        ```
    *   **启动并启用服务**:
        ```bash
        sudo systemctl daemon-reload
        sudo systemctl start chaintrace
        sudo systemctl enable chaintrace
        ```
    *   **检查服务状态**:
        ```bash
        sudo systemctl status chaintrace
        ```

## 3. 前端应用部署 (Frontend)

前端是一个纯静态应用，可以部署在任何Web服务器上（如Nginx）。

### **3.1. 构建生产版本**
1.  **环境准备**: 在您的开发机（或任何安装了Node.js和npm的机器）上，进入前端代码的根目录。
2.  **安装依赖**:
    ```bash
    npm install
    ```
3.  **构建应用**:
    ```bash
    npm run build
    ```
    此命令执行成功后，会在项目根目录下生成一个 `dist` 文件夹。这个文件夹包含了部署所需的全部文件。

### **3.2. 部署到 Nginx**
1.  **拷贝文件**: 将您本地 `dist` 文件夹中的**所有内容**，拷贝到Web服务器的指定目录（例如: `/var/www/chaintrace/`）。
2.  **配置 Nginx**:
    *   **创建Nginx配置文件**:
        ```bash
        sudo nano /etc/nginx/sites-available/chaintrace
        ```
    *   **粘贴以下配置** (请根据您的实际情况修改 `server_name` 和 `root`):
        ```nginx
        server {
            listen 80;
            server_name chaintrace.yourcompany.com; # 您的域名或IP

            root /var/www/chaintrace;
            index index.html;

            location / {
                # 这是支持React Router的关键配置
                try_files $uri $uri/ /index.html;
            }
        }
        ```
    *   **启用站点并重启Nginx**:
        ```bash
        sudo ln -s /etc/nginx/sites-available/chaintrace /etc/nginx/sites-enabled/
        sudo nginx -t  # 测试配置是否正确
        sudo systemctl restart nginx
        ```
3.  **安全增强**: 强烈建议为您的前端站点配置 **HTTPS**。您可以使用 Let's Encrypt 等工具免费获取SSL证书。

## 4. 最终配置与使用

1.  **用户访问**: 团队成员现在可以通过浏览器访问您配置的前端域名（例如 `http://chaintrace.yourcompany.com`）。
2.  **配置代理**: 在登录页面，用户需要在“代理 API 地址”输入框中，填入**后端代理服务器的地址**（例如 `http://<后端服务器IP>:8001`）。
3.  **登录**: 使用管理员在系统中创建的账户登录。初始默认账户为 `admin` / `admin`，**请务必在首次登录后立即修改密码！**

---

## 5. 维护与备份

*   **备份 (极其重要!)**:
    *   **必须**定期备份后端服务器上的 **`chaintrace.db`** 文件。这个文件包含了所有的设备历史、用户数据和审计日志，是整个系统的核心。
    *   同时，备份 `config.ini` 文件也同样重要。
*   **更新**:
    *   **后端**: 停止`chaintrace`服务 -> 拷贝新代码 -> 在虚拟环境中运行 `pip install -r requirements.txt` -> 重启`chaintrace`服务。
    *   **前端**: 在开发机上 `npm run build` -> 将新的 `dist` 文件夹内容拷贝到Web服务器覆盖旧文件。
