## 第一部分：绪论 (背景与意义)

### P1：封面
*   **主标题**：基于区块链与AI的智能网络配置审计系统设计与实现
*   **副标题**：—— “链踪”：网络配置的智能守护者
*   **信息**：学院、专业、姓名、学号、导师、日期

### P2：研究背景与痛点 (Why?)
*   **核心痛点**：
    *   **高风险**：一次错误的配置可能导致全网瘫痪（“蝴蝶效应”）。
    *   **黑盒操作**：SSH操作过程不透明，谁做的？做了什么？难以追溯。
    *   **事后诸葛亮**：传统审计是被动的，只有出事了才去查日志，无法“事前预防”。
*   **现有方案不足**：
    *   **Ansible**：侧重“执行”，缺乏严谨的审计链。
    *   **Rancid**：侧重“备份”，缺乏实时性和智能化。

### P3：研究目标与核心理念 (What?)
*   **核心理念**：
    1.  **区块链思想**：构建不可篡改的“审计链”，解决信任问题。
    2.  **AI赋能**：引入LLM大模型，实现从“事后追溯”到**“事前治理”**的飞跃。
*   **系统定位**：一个集“不可变记录、事前智能治理、安全交互终端”于一体的企业级协作平台。

## 第三部分：核心创新与实现 (重点)

### P6：创新点一：不可变审计链 (The Chain)
*   **实现原理**：
    *   基于 SHA-256 哈希算法。
    *   `Current_Hash = Hash(Data + Timestamp + Prev_Hash)`。
    *   环环相扣，任何对历史数据的篡改都会导致链条断裂。
*   **功能展示**：
    *   链完整性验证（可视化验证过程）。
    *   智能回滚（基于不可变历史生成回滚方案）。

### P7：创新点二：AI驱动的“事前治理” (The Gatekeeper)
*   **核心逻辑**：**先审计，后执行**。
*   **工作流**：
    1.  用户提交配置 -> 2. AI合规引擎介入。
    3.  基于自然语言策略（如“禁止使用Telnet”）进行审计。
    4.  **通过** -> 写入区块链；**失败** -> 直接拦截请求。
*   *配图：论文中的“图4-1 AI事前治理变更拦截工作流图”*

### P8：创新点三：法证级安全交互终端 (The Secure Console)
*   **痛点解决**：解决了Web端操作体验差、无法实时管控的问题。
*   **技术亮点**：
    *   **原生体验**：支持 `Tab` 补全，实时回显。
    *   **协议级拦截**：后端实时监控输入流，检测到高危命令（如 `write memory`）时，发送 `CTRL+U` 信号物理清除输入行。
    *   **审计闭环**：“断连自动审计”机制，防止恶意关闭浏览器逃避记录。

### P9：创新点四：企业级架构特性
*   **可插拔AI驱动**：适配器模式设计，修改配置即可切换 Gemini/Spark/DeepSeek。
*   **原子化权限**：支持对 Operator 进行精细化的权限下放（如仅授予 `device:restart`）。
*   **安全写入启动配置**：二次验证 + 令牌机制，确保持久化操作的严肃性。
