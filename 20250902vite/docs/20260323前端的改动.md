# 2026-03-23 改动记录

## 概述

本次改动在完成 JWT 安全加固后，修复了多个运行时 Bug，并对前端浅色主题进行了全面重设计。

---

## Bug 修复

### 1. 修复设备状态轮询端点 500 错误
**文件：** `agentv2/api_routes.py`

**问题：** `GET /api/devices/poll-status` 返回 500，日志报 `NameError: name 'models' is not defined`。

**原因：** 新增的 `poll_all_device_statuses` 函数直接使用了 `models.Device`，但文件顶部未导入 `models` 模块。

**修复：** 在 `import crud` 之后添加 `import models`。

---

### 2. 修复添加新设备 422 错误
**文件：** `agentv2/core.py`、`agentv2/crud.py`

**问题：** 点击"添加设备"提示 `422 Unprocessable Content`。

**原因：** 前端发送 `tags: ["tag1", "tag2"]`（JSON 数组），但后端 `DevicePayload` 定义的是 `tags: Optional[str]`（字符串），类型不匹配导致 FastAPI 验证失败。

**修复：**
- `core.py`：将 `DevicePayload.tags` 改为 `Optional[List[str]] = []`
- `crud.py`：在 `create_device_with_genesis_block()` 和 `update_device()` 中，存入数据库前用 `','.join(tags)` 将列表转为逗号分隔字符串

---

### 3. 修复保存会话 422 错误
**文件：** `agentv2/core.py`

**问题：** 点击"保存会话并审计"返回 `422 Unprocessable Content`。

**原因：** 前端发送的请求体只包含 `{ sessionId }`，但 `AuditTriggerPayload` 中 `operator` 字段为必填项，导致验证失败。实际上 `operator` 信息已通过 JWT token 由 `actor` 参数提供，无需客户端重复传递。

**修复：** 将 `AuditTriggerPayload.operator` 改为 `Optional[str] = None`。

---

### 4. 修复 WebSocket 断开连接时的级联错误
**文件：** `agentv2/websocket_handler.py`

**问题：** 连接离线设备时，后端日志出现大量级联异常（`ConnectionClosedOK` → `ClientDisconnected` → `WebSocketDisconnect` → `RuntimeError: Cannot call "send" once a close message has been sent`），干扰日志可读性。

**原因：** SSH 连接失败后，前端 WebSocket 已关闭，但后端 `except` 块仍尝试往已关闭的连接发送错误消息，以及 `finally` 块再次调用 `websocket.close()` 导致连锁报错。

**修复：**
- 将 `except` 块中的 `websocket.send_text()` 包裹在 `try/except` 中，静默处理已断开连接的情况
- 将 `finally` 块中的 `websocket.close()` 同样包裹在 `try/except` 中

---

### 5. 新增模拟终端后撤销（已移除）
**文件：** `agentv2/websocket_handler.py`

**背景：** 为模拟模式新增了 `simulated_terminal_handler()` 虚拟终端函数，但由于模拟终端无法支持 `conf t` 等配置命令，且保存会话等完整工作流无法正常运行，经用户决定放弃模拟终端功能，已将相关代码全部移除。WebSocket 处理器恢复为仅走真实 SSH 连接路径。

---

## 功能改进

### 6. 浅色主题全面重设计
**文件：** `src/index.css`、`src/components/SettingsModal.tsx`

**问题：** 所有浅色主题存在以下通用问题：
- 按钮默认背景（`bg-bg-700`）过浅，与白色卡片几乎无法区分
- 悬停色（`hover:bg-bg-600`）比默认状态更浅，悬停反馈几乎不可见
- `border-white/10` 边框在浅色背景下完全不可见
- 部分文字对比度不足

**修复方案：** 将原有4个自制浅色主题替换为业界成熟的主流方案，并统一视觉层次（bg-800 = 按钮默认背景，bg-700 = 悬停背景，bg-600 = 输入框背景）。

| 旧主题名 | 新主题名 | 来源 |
|---------|---------|------|
| 北极星 (polaris) | **GitHub Light** | GitHub Primer 官方设计系统 |
| 浅空蓝 (sky) | **Catppuccin Latte** | 2024年最流行开发者主题 |
| 薄荷绿 (mint) | **One Light** | Atom 编辑器经典主题（已被移植到数十款工具） |
| 薰衣草紫 (lavender) | **Solarized Light** | 2011年诞生的永恒经典 |

**额外修复：** 为所有浅色主题添加 `border-white/10` 和 `border-white/20` 的 CSS 覆盖规则，将不可见的白色透明边框替换为主题色的半透明变体。
