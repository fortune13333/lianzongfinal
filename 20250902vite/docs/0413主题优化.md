# 0413 主题优化改动记录

## 一、全局 Tab 栏重构（App.tsx）

**背景**：原来的终端标签页管理在 `TerminalContainer` 内部，与顶部导航割裂，用户切换视图时终端 SSH 连接会断开。

**改动**：
- 在 `App.tsx` 顶层新增始终可见的 Tab 栏，左侧为固定视图标签（受管设备 / 网络拓扑），右侧为动态设备终端标签
- `TerminalContainer` 改为 CSS `hidden` 隐藏而非卸载，保留 WebSocket/SSH 连接
- 新增两个 `useEffect`：标签全关时自动切回 dashboard；新标签打开时自动切换到 terminal 视图
- `MainView` 类型扩展为 `'dashboard' | 'topology' | 'terminal'`

**修改文件**：`src/App.tsx`、`src/components/TerminalContainer.tsx`

## 二、终端高度修复（InteractiveTerminal.tsx）

**问题**：xterm 终端底部有大片黑色空白，未填满容器。

**原因**：`flex-grow` + `h-full` 无法给 xterm 提供确定尺寸。

**修复**：
- 终端容器改为 `relative overflow-hidden min-h-0`
- 内部 `div` 改为 `absolute inset-0`，xterm 挂载到绝对定位容器，尺寸由父级决定

**修改文件**：`src/components/InteractiveTerminal.tsx`

## 三、17 个预设主题（index.css + SettingsModal.tsx）

### 3.1 设计原则

- **暗色主题**：文字统一使用中性 zinc 色（不随主题变色），主题个性体现在背景色和 primary 配色
- **亮色主题**：文字统一使用深灰色，背景和 primary 配色区分主题
- 杜绝"颜色滤镜"感：文字绝不跟随主题大幅变色

### 3.5 SettingsModal 外观设置重构

- 原有多个独立下拉选择器改为卡片网格 UI
- 每张卡片显示 3 个色板圆点 + 主题名称，选中时显示 ✓

- 保留"自定义配色"可折叠区域供进阶用户使用
- 预设选择只写入 `bgTheme`，不再写 `theme`

**修改文件**：`src/index.css`、`src/components/SettingsModal.tsx`

## 四、动态极光背景（App.tsx + index.css）

### 4.1 整体架构

所有暗色预设主题（含原有 `deep-space`，共 12 个）启用动态极光背景。亮色主题不渲染光球。

**关键设计**：`BackgroundLayers` 组件定义在 `App` 函数**外部**（模块级），并用 `React.memo` 包裹，接收 `bgTheme` prop。

> 若定义在 `App` 内部，每次 App 重渲染（每 5s fetchData 一次）都会产生新函数引用，导致 React 卸载并重新挂载组件，CSS 动画从 0% 强制重置，形成周期性卡顿。

### 4.2 光球颜色（THEME_BLOBS）

每个主题配置 3 个 RGBA 颜色，与该主题的 primary 配色呼应：

```ts
const THEME_BLOBS: Record<string, [string, string, string]> = {
  'deep-space':    ['rgba(147,51,234,0.40)', 'rgba(37,99,235,0.34)',  'rgba(6,182,212,0.30)'],
  'proton-purple': ['rgba(139,92,246,0.48)', 'rgba(79,70,229,0.38)',  'rgba(45,212,191,0.28)'],
  // ... 其余 10 个主题
};
```

### 4.3 公转动画实现

**原理**：零尺寸 wrapper 锚定屏幕中心，`rotate(θ) translateX(R)` 驱动光球绕屏幕中心公转。

```
wrapper: position absolute, left: 50%, top: 50%（零尺寸）
  transform-origin 默认 50% 50% = 屏幕中心点
  动画: rotate(θ) translateX(R)
    → 先沿局部 X 轴移动 R
    → 再绕屏幕中心旋转
  → 光球中心以屏幕中心为圆心、R 为半径做圆周运动

blob child: position absolute, 负边距居中于 wrapper
  width/height: ~55vw（大光球）
  backgroundColor: 主题色
```

**三球均布方案**：相同周期（35s）+ 初始角均匀错开 120°，保证三球始终 120° 间隔，背景各方向亮度均匀。

```css
@keyframes aurora-screen-orbit-1 {
  from { transform: rotate(0deg)   translateX(22vw); }
  to   { transform: rotate(360deg) translateX(22vw); }
}
@keyframes aurora-screen-orbit-2 {
  from { transform: rotate(120deg)  translateX(28vw); }
  to   { transform: rotate(480deg)  translateX(28vw); }
}
@keyframes aurora-screen-orbit-3 {
  from { transform: rotate(240deg)  translateX(16vw); }
  to   { transform: rotate(600deg)  translateX(16vw); }
}

.aurora-orbit-1 { will-change: transform; animation: aurora-screen-orbit-1 35s linear infinite; }
.aurora-orbit-2 { will-change: transform; animation: aurora-screen-orbit-2 35s linear infinite; }
.aurora-orbit-3 { will-change: transform; animation: aurora-screen-orbit-3 35s linear infinite; }
```

### 4.4 无卡顿保证

- `linear + 0°→360°`：起止位置数学上完全相同，不存在"复位"跳变
- `will-change: transform`：GPU 合成器线程处理，不占用 JS 主线程
- `filter: blur(90px)`：大模糊半径，GPU 自动提升为合成层

### 4.5 各轨道参数

| | 轨道半径 | 初始角 | 光球尺寸 |
|-|---------|--------|---------|
| Blob 1 | 22vw | 0° | 60vw |
| Blob 2 | 28vw | 120° | 60vw |
| Blob 3 | 16vw | 240° | 60vw |
