# “链踪”V7.1 功能说明：部署历史

## 1. 功能概述：从“临时通知”到“永久档案”

在 V7.1 版本之前，执行“批量部署模板”等可能耗时较长的操作时，系统会通过一个临时的 `toast` 消息来提示操作结果。然而，这种方式存在一个明显的痛点：

*   **信息易失**: 如果操作员在部署期间切换了窗口或暂时离开，很容易错过最终的通知。
*   **缺乏追溯**: 一旦通知消失，便无法再回头查看某次特定部署任务的详细结果，例如，具体哪几台设备部署失败了，以及失败的原因是什么。

为了解决这个问题，V7.1 版本引入了一个全新的、至关重要的企业级功能——**部署历史 (Deployment History)**。

它的核心目标是：将每一次批量部署任务，都作为一个**永久性的、不可篡改的“任务档案”**记录下来。这为管理员提供了一个完整的、可随时查阅的部署操作审计日志，确保了操作的完全透明度和可追溯性。

---

## 2. 核心价值

此功能为“链踪”平台带来了三大核心价值提升：

1.  **操作的完全透明度**:
    您再也无需猜测部署任务的最终结果。每一次部署，无论成功、失败还是部分失败，其完整的执行记录都会被系统捕获并永久保存。

2.  **事后审计与深度追溯**:
    您可以轻松地回答以下关键的管理问题：
    *   “上周五是谁向核心交换机部署了哪个模板？”
    *   “某次部署任务中，影响了哪些设备？”
    *   “为什么那次部署会有三台设备失败了？”

3.  **高效的故障诊断**:
    当一次批量部署出现部分失败时，“部署历史”功能的最大优势便体现出来。您无需重新执行整个任务，只需展开失败任务的详情，即可看到**每一台失败设备的具体错误信息**（例如“连接超时”、“认证失败”或“在config.ini中未找到”）。这使得故障排查变得极为精准和高效。

---

## 3. 使用方法

部署历史的记录是全自动的，您只需像往常一样操作即可。

### 第一步：正常执行批量部署

1.  在**仪表盘**，勾选您想要部署的设备。
2.  点击底部工具栏的 **“部署模板”**。
3.  在弹出的窗口中，选择模板并确认部署。
4.  系统会在后台自动执行部署，并为您创建一条详细的部署历史记录。

### 第二步：查阅部署历史

1.  以 `admin` 用户登录，进入**管理中心**。
2.  切换到新增的 **“部署历史”** 选项卡。

### 第三步：分析部署结果

您会看到一个包含所有历史部署任务的列表，其中列出了：
*   **时间**: 任务执行的精确时间。
*   **操作员**: 执行该任务的用户。
*   **模板**: 本次部署所使用的模板名称。
*   **状态**: 任务的总体结果（“成功”或“部分失败”）。
*   **结果摘要**: 一个简短的总结，如“5 成功, 2 失败”。

### 第四步：深入查看详情

对于任何一条记录，点击最右侧的 **“查看详情”** 链接。

*   该条目会向下展开，显示一个详细的设备列表。
*   列表中会清晰地标出**每一台设备**的部署状态，并附上具体的成功信息（例如，“部署成功，新版本为 12”）或失败原因。

**总结**: 部署历史功能的引入，将“链踪”的自动化能力提升到了一个新的、更可靠、更专业的层次。它为关键操作提供了坚实的审计基础，是项目迈向成熟企业级治理平台的又一关键里程碑。
