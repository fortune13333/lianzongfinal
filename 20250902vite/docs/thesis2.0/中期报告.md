## 一、 课题简介

本课题旨在解决传统网络配置管理中存在的变更不透明、治理滞后、操作风险高及审计困难等痛点。通过结合**区块链的不可变性思想**与**人工智能（AI）大语言模型的逻辑推理能力**，设计并实现一个名为“链踪”的智能网络配置审计系统。该系统不仅提供基于哈希链的不可篡改的配置历史账本，还引入了AI驱动的“事前治理”引擎、实时安全交互终端以及“断连自动审计”等创新机制，实现了网络配置管理从“事后追溯”到“事前防范”的模式转变。

## 二、 前期工作简述及目前进度

自开题以来，在导师的指导下，本项目已按计划完成了需求分析、系统架构设计以及核心功能模块的开发，目前系统已达到功能高度完备的企业级产品原型阶段。具体完成的工作如下：

### 1. 系统总体架构与数据库设计
*   **前后端分离架构落地**：前端采用 React 18 + TypeScript + Tailwind CSS 构建了现代化单页应用；后端采用 Python FastAPI 框架构建了高性能的异步 API 服务。
*   **数据库模型建立**：完成了基于 SQLite 的关系型数据库设计，使用 SQLAlchemy ORM 实现了用户、设备、区块（审计链）、策略等多表关联的 E-R 模型，取代了初期的 JSON 文件存储，大幅提升了数据一致性与并发处理能力。

### 2. 核心功能模块开发
*   **不可变审计链机制**：成功实现了基于 SHA-256 算法的区块哈希计算与链式追加逻辑。每一次配置变更都会生成包含前一区块哈希的不可篡改记录，并实现了前端可视化的“链完整性一键验证”功能。
*   **AI 事前治理工作流**：构建了“策略即代码”模块，允许使用自然语言定义合规规则。在配置保存的核心数据流中，强制接入 AI 进行合规性审查，实现了对违规变更的实时拦截。
*   **安全交互式 Web 终端**：利用 Xterm.js 和 WebSocket 技术，结合后端的 Netmiko 库，实现了浏览器端到真实网络设备的实时 SSH 通道，支持原生 CLI 体验（如 Tab 键补全）。
*   **企业级管理功能**：实现了基于角色的原子化权限管理（RBAC）、动态配置模板批量部署以及详尽的系统操作审计日志。

## 三、 已解决的关键技术问题

在开发过程中，重点攻克了以下几个核心技术难点，构建了系统的“法证级”安全模型：

1.  **高危命令的实时拦截**：摒弃了早期不可靠的拦截方案，在后端协议层实现了基于 `config.ini` 规则驱动的拦截引擎。在用户按下回车瞬间，精准识别违规命令（如 `write memory`）并向设备发送 `CTRL+U` 原子清除信号，彻底杜绝了危险命令的执行。
2.  **终极审计闭环（断连自动审计）**：针对用户恶意退出或网络意外断开导致审计记录丢失的漏洞，实现了“断连自动审计”机制。后端能智能跟踪“脏”会话，在断连时自动重连设备抓取最终状态，并作为“自动快照”强制上链打上“耻辱烙印”，确保审计无死角。
3.  **可插拔 AI 驱动架构**：为了解耦核心业务与底层 AI 模型，设计了适配器模式的 `ai_drivers` 模块。目前已支持通过修改配置文件一键切换讯飞星火（Spark）和 Google Gemini 等不同 AI 引擎，极大提升了系统的灵活性与可扩展性。

## 四、 目前存在的问题及解决方案

尽管核心功能已基本完善，但在深度测试和实际模拟部署中，仍发现以下待优化的问题：

1.  **AI 响应延迟与并发瓶颈**：在进行复杂的合规性审计或长文本配置分析时，调用外部 AI API 存在一定的网络延迟；同时多设备并发审计时可能触发 API 速率限制。
    *   *解决方案*：在后端引入异步任务队列，优化提示词（Prompt）以减少不必要的 Token 消耗；完善“指数退避重试”机制以增强容错性；前端增加更友好的分步异步状态提示（ProgressToast），缓解用户等待焦虑。
2.  **多厂商设备兼容性细节**：虽然 Netmiko 提供了良好的抽象，但在某些特定老旧型号或特殊厂商设备上，交互式终端的提示符识别和回显处理偶尔会出现轻微错位。
    *   *解决方案*：针对主流厂商（如 Cisco, Huawei, H3C）建立更详细的测试用例库，在后端 `unified_io_handler` 中增加针对特定厂商终端特性的正则微调和异常捕获。
3.  **UI/UX 细节打磨**：深浅色主题切换及部分复杂模态框在小屏幕设备上的响应式适配仍有瑕疵。
    *   *解决方案*：依据 Tailwind CSS 规范进行全面的响应式断点审查，优化移动端或小尺寸窗口下的表格和终端视图展示。
