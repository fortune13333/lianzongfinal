## 一、功能概述

本次新增"网络拓扑可视化"功能，实现了：

1. **自动拓扑发现**：通过 SSH 登录受管设备，执行 `show cdp neighbors detail` / `show lldp neighbors detail` 命令，解析邻居关系并持久化到数据库。
2. **模拟演示模式**：无需真实设备，一键加载预设模拟数据，用于演示和测试。
3. **前端可视化**：使用 React Flow（`@xyflow/react`）在独立的"网络拓扑"标签页中以图形化方式展示节点与连接关系。

### 2.1 数据库模型

**文件：`agentv2/models.py`**

新增 `TopologyLink` ORM 模型，表示一条有向邻居关系（由某台受管设备报告的单条 CDP/LLDP 邻居记录）：

```python
class TopologyLink(Base):
    __tablename__ = "topology_links"
    id = Column(String, primary_key=True, default=lambda: str(uuid.uuid4()))
    source_device_id = Column(String, nullable=False, index=True)  # 报告该邻居的受管设备ID
    source_port = Column(String, nullable=True)    # 本地端口，如 GigabitEthernet0/0
    target_device_id = Column(String, nullable=False)              # 邻居主机名（大写）
    target_port = Column(String, nullable=True)    # 邻居端口，如 GigabitEthernet1/0/1
    target_ip = Column(String, nullable=True)      # 邻居管理IP
    target_platform = Column(String, nullable=True)# 邻居平台型号
    protocol = Column(String, nullable=False, default='cdp')  # 'cdp' | 'lldp'
    discovered_at = Column(DateTime(timezone=True), server_default=func.now())
```

### 2.4 服务层（SSH 发现逻辑）

**文件：`agentv2/services.py`**（末尾追加）

#### CDP 解析器 `_parse_cdp_neighbors(raw, source_device_id)`

逐行解析 `show cdp neighbors detail` 的输出。关键逻辑：

- `Device ID:` 行：开始新的邻居记录，并截断域名后缀（`SW1.corp.local` → `SW1`），统一转大写
- `IP address:` 行：正则提取管理 IP
- `Platform:` 行：提取平台型号（逗号前的部分）
- `Interface:` 行：正则同时提取本地端口和邻居端口

```python
# 关键正则示例
m = re.search(r"Interface:\s*(\S+),\s*Port ID[^:]*:\s*(\S+)", line)
```

#### LLDP 解析器 `_parse_lldp_neighbors(raw, source_device_id)`

逐行解析 `show lldp neighbors detail`，字段映射：

| LLDP 输出行 | 映射字段 |
|------------|---------|
| `System Name:` | `target_device_id` |
| 纯 IP 地址行 | `target_ip`（第一个出现的） |
| `System Description:` | `target_platform`（截取前64字符） |
| `Local Intf:` | `source_port` |
| `Port ID:` | `target_port` |

#### SSH 入口函数 `perform_discover_topology(device_id)`

```python
def perform_discover_topology(device_id: str) -> List[Dict[str, Any]]:
    device_info = get_device_info(device_id)  # 从DB取设备连接信息
    with ConnectHandler(**device_info) as conn:
        cdp_raw = conn.send_command("show cdp neighbors detail", read_timeout=15)
        links = _parse_cdp_neighbors(cdp_raw, device_id)
        if not links:  # CDP 无结果则尝试 LLDP
            lldp_raw = conn.send_command("show lldp neighbors detail", read_timeout=15)
            links = _parse_lldp_neighbors(lldp_raw, device_id)
    return links
```

**异常处理**：
- `NetmikoAuthenticationException` → HTTP 401
- `NetmikoBaseException` → HTTP 504（网络/超时错误）

      "position": { "x": 0, "y": 0 }

**节点 `managed` 字段**：通过对比 `target_device_id` 是否存在于受管设备列表（`db.query(models.Device)`）来标注，用于前端区分颜色。

#### `POST /api/topology/discover` 请求体

```json
{
  "device_ids": ["RTR01-NYC", "RTR02-LA"],  // 可选，null 则发现所有受管设备
  "simulation": false                        // true 时跳过 SSH，使用预设模拟数据
}
```

**模拟数据**（4条预设链路，构成一个简单的 2地区 4节点拓扑）：
- RTR01-NYC → SW1-NYC（CDP）
- RTR01-NYC → RTR02-LA（CDP，模拟广域链路）
- SW1-NYC → SW2-NYC（CDP）
- RTR02-LA → SW1-LA（CDP）

**部分失败处理**：对每台设备单独 try/catch，失败的设备记录到 `errors` 列表后继续下一台。全部失败才返回 504，部分失败返回 200 并在响应体的 `errors` 字段中附带失败信息。

#### 圆形布局算法

首次渲染时，将后端返回的 `position: {x:0, y:0}` 替换为圆形布局坐标：

```typescript
function circularLayout(nodes: TopologyNode[]): TopologyNode[] {
  const cx = 500, cy = 350;
  const radius = Math.min(350, 80 * nodes.length);
  return nodes.map((n, i) => {
    const angle = (2 * Math.PI * i) / nodes.length - Math.PI / 2;
    return { ...n, position: { x: cx + radius * Math.cos(angle), y: cy + radius * Math.sin(angle) } };
  });
}
```

半径随节点数量动态调整（最大 350px），从 12 点钟方向顺时针排列。

#### 边标签格式化

将冗长的接口名缩写，使边标签简洁可读：

```typescript
function formatEdgeLabel(data: TopologyEdge['data']): string {
  const abbrev = (port) => port
    .replace(/GigabitEthernet/i, 'Gi')
    .replace(/FastEthernet/i, 'Fa')
    .replace(/TenGigabitEthernet/i, 'Te');
  return `${abbrev(sourcePort)} ↔ ${abbrev(targetPort)}`;
  // 例：GigabitEthernet0/1 ↔ GigabitEthernet1/0/1 → Gi0/1 ↔ Gi1/0/1
}
```

**边颜色区分协议**：
- CDP 链路：绿色（`#34d399`）
- LLDP 链路：蓝色（`#60a5fa`）

## 七、使用说明

1. **模拟演示**（无需设备）：打开"网络拓扑"标签页 → 点击"模拟演示"，即可看到预设的4节点拓扑图
2. **真实发现**（需要受管设备）：先在"受管设备"中添加设备，然后切换到"网络拓扑" → 点击"开始发现"，系统会逐台 SSH 登录并执行 CDP/LLDP 命令
3. **交互操作**：支持拖拽节点、滚轮缩放、小地图导航；节点位置可自由调整（本次布局算法只决定初始位置）
4. **刷新**：点击"刷新"从数据库重新加载最新数据（不重新 SSH）
5. **清除**：需要 `system:reset` 权限（admin 角色），清除后画布恢复空状态
