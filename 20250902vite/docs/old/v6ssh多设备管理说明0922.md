# “链踪”V7.5 功能说明：多凭证 SSH 管理

欢迎！本次更新为“链踪”项目引入了一项至关重要的企业级功能——**多凭证 SSH 管理**。这项升级允许您为不同的网络设备或设备组配置各自独立的登录凭证，从而极大地提升了系统的安全性、灵活性和在真实运维场景中的实用性。

---

## 1. 引言：为何需要多凭证管理？

在之前的版本中，`config.ini` 文件中只有一个全局的 `[credentials]` 区段。这意味着所有被管理的设备都必须使用同一套用户名和密码。

在真实的、稍具规模的企业网络中，这种“一把钥匙开所有门”的方式是**不可想象的，也是极不安全的**。

*   **安全最佳实践**: 遵循“最小权限原则”，不同区域、不同级别的设备（如核心层 vs. 接入层，DMZ区 vs. 办公区）理应使用不同的凭证。
*   **运维现实**: 不同厂商的设备、不同历史时期上线的设备，往往有不同的管理账号。
*   **多团队协作**: 在大型组织中，不同团队可能只拥有部分设备的管理权限。

本次更新正是为了解决这一核心局限，使“链踪”能更好地适应复杂且安全要求严格的真实世界网络环境。

---

## 2. 核心变更：`config.ini` 格式升级

为了实现多凭证管理，我们对 `config.ini` 的配置结构进行了一次简单而强大的升级。

### 旧版 `config.ini` (单一凭证)

```ini
[credentials]
username = admin
password = cisco123

[device_map]
; 格式: 设备ID = IP地址, 设备类型
CORE-SW-01 = 10.1.1.1, cisco_ios
ACCESS-SW-01 = 10.2.1.1, huawei_vrp
```

### **新版 `config.ini` (多凭证)**

```ini
; 您可以定义多个凭证“配置文件”，名称可自定义
[credentials_cisco_core]
username = core_admin
password = VeryComplexPassword!

[credentials_huawei_access]
username = access_user
password = Huawei@123

[device_map]
; 格式: 设备ID = IP, 设备类型, [凭证配置名称]
CORE-SW-01 = 10.1.1.1, cisco_ios, credentials_cisco_core
ACCESS-SW-01 = 10.2.1.1, huawei_vrp, credentials_huawei_access
```

如您所见，核心变化有两点：
1.  您可以创建**任意多个**以 `[credentials_...]` 命名的凭证区段。
2.  在 `[device_map]` 的每一行末尾，增加**第三个参数**，用于指定该设备应使用哪个凭证区段。

## 3. 分步配置指南

配置过程非常简单，只需两步即可完成。

### 第 1 步：定义您的凭证“配置文件”

1.  打开您的 `config.ini` 文件。
2.  移除或重命名旧的 `[credentials]` 区段。
3.  根据您的需求，创建新的凭证区段。区段名称可以任意取，但为了清晰起见，我们**强烈建议**以 `credentials_` 作为前缀（例如 `[credentials_firewalls]` 或 `[credentials_dc_switches]`）。
4.  在每个区段下，填写对应的 `username` 和 `password`。

**示例：**
```ini
[credentials_cisco_routers]
username = router_admin
password = S3cureR0uterP@ss

[credentials_h3c_switches]
username = switch_op
password = H3C_Password123
```

### 第 2 步：将设备映射到凭证

1.  找到文件中的 `[device_map]` 区段。
2.  对于每一行设备条目，在末尾添加一个逗号 (`,`)，然后紧跟您希望该设备使用的**凭证区段的完整名称**。

**示例：**
```ini
[device_map]
; ... 其他注释 ...
RTR01-NYC = 192.168.1.1, cisco_ios, credentials_cisco_routers
SW01-SFO = 10.10.5.254, cisco_ios, credentials_cisco_routers
H3C-ACCESS-01 = 10.10.10.1, hp_comware, credentials_h3c_switches
```
-   `RTR01-NYC` 和 `SW01-SFO` 将会使用 `[credentials_cisco_routers]` 区段的凭证进行登录。
-   `H3C-ACCESS-01` 则会使用 `[credentials_h3c_switches]` 的凭证。

### 第 3 步：重启代理

**保存**您对 `config.ini` 文件的修改，然后**重启 `agent.py`** (或 `agent.exe`)。新的凭证配置将立即生效。

## 4. 常见问题 (FAQ)

**问：`[device_map]` 中的第三个参数（凭证区段名称）是必需的吗？**
**答：** 是的。从V7.5版本开始，为了确保配置的明确性，每一条设备映射都必须显式指定其使用的凭证区段。

**问：如果我指定的凭证区段名称在文件中不存在，会发生什么？**
**答：** 代理程序在启动时或尝试连接该设备时，会在控制台打印一条明确的错误日志，指出找不到对应的凭证区段，并且连接会失败。这可以帮助您快速定位配置错误。

**问：模拟模式 (`username = SIM_USER`) 还能用吗？**
**答：** 可以！新的 `is_simulation_mode()` 检查会扫描**所有**以 `credentials` 开头的区段。只要其中**任意一个**区段的 `username` 被设置为 `SIM_USER`，整个代理就会进入模拟模式。
