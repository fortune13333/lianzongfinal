## 改进 1：PostgreSQL 双模式数据库支持

### 设计思路
不破坏现有开发体验（SQLite 零配置启动），通过 `DATABASE_URL` 环境变量无缝切换到 PostgreSQL。`_is_sqlite` 标志在内部控制仅 SQLite 需要的 `check_same_thread` 参数。

### 文件改动

#### `agentv2/database.py`（核心重写）
```python
import os
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker, declarative_base

DATABASE_URL = os.environ.get("DATABASE_URL", "sqlite:///./chaintrace.db")
_is_sqlite = DATABASE_URL.startswith("sqlite")
_connect_args = {"check_same_thread": False} if _is_sqlite else {}

engine = create_engine(DATABASE_URL, connect_args=_connect_args)
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
Base = declarative_base()
```
删除了原有的硬编码 SQLite 路径，`init_db()` 和 `get_db()` 函数保持不变。

## 改进 2：LDAP / Active Directory 认证

### 设计思路
- **双轨认证**：LDAP 配置存入数据库 `settings` 表，管理员可随时启用/禁用，未启用时完全退回本地密码验证。
- **首次自动建档**：LDAP 验证通过后，若本地 `users` 表无该账号则自动创建，角色默认 `operator`，管理员在"用户管理"中可调整权限。
- **License 门控**：`check_feature("ldap")` 确保此功能仅专业版 / 企业版可用。

```python
def verify_ldap_credentials(username: str, password: str, ldap_cfg: dict) -> bool:
    """两步绑定验证：先用 bind_dn 搜索用户 DN，再用用户 DN + 密码二次绑定。"""
    from ldap3 import Server, Connection, ALL
    try:
        server = Server(ldap_cfg["server"], port=ldap_cfg.get("port", 389),
                        use_ssl=ldap_cfg.get("use_ssl", False), get_info=ALL)
        conn = Connection(server, user=ldap_cfg["bind_dn"],
                          password=ldap_cfg["bind_password"], auto_bind=True)
        search_filter = ldap_cfg.get("user_search_filter",
                            "(sAMAccountName={username})").format(username=username)
        conn.search(ldap_cfg["base_dn"], search_filter, attributes=["distinguishedName"])
        if not conn.entries:
            return False
        user_dn = conn.entries[0].entry_dn
        user_conn = Connection(server, user=user_dn, password=password, auto_bind=True)
        return user_conn.bound
    except Exception as e:
        logging.warning(f"LDAP auth error: {e}")
        return False


def get_or_create_ldap_user(db: Session, username: str) -> models.User:
    """LDAP 认证成功后确保本地有对应记录，首次登录自动创建 operator 账号。"""
    user = crud.get_user_by_username(db, username)
    if not user:
        user = models.User(
            username=username,
            password=get_password_hash(secrets.token_hex(32)),  # 随机密码，永不使用
            role="operator",
            extra_permissions="",
        )
        db.add(user)
        db.commit()
        db.refresh(user)
    return user
```

## 改进 3：PDF 审计报告导出

### 设计思路
- 使用 **WeasyPrint** 将 Jinja2 渲染的 HTML 转为 PDF，完全在服务端完成，不依赖浏览器打印。
- 两份报告独立端点：审计日志报告（按时间范围）和区块链合规报告（按设备）。
- WeasyPrint 依赖 Linux 系统库，在 Dockerfile 中安装；本地 Windows 开发环境不安装时，端点返回 HTTP 503 友好提示而非崩溃。

## 改进 4：离线 RSA License 机制

### 设计思路
- **公私钥分离**：2048-bit RSA 密钥对，私钥由开发者保管（不进 git），公钥嵌入 `license.py` 随产品发布。
- **签名格式**：`.lic` 文件为 JSON，包含 `payload`（Base64 编码的 JSON 明文）和 `signature`（Base64 编码的 PKCS1v15+SHA256 签名）。
- **内容字段**：`customer`、`max_devices`、`features`（列表）、`expires_at`、`issued_at`。
- **缓存机制**：`_cached_license` 全局变量，进程内只验签一次，避免每次请求重复 I/O。
- **无 License = 社区版**：缺少 `.lic` 文件或签名验证失败均退化为社区版（最多 5 台设备，无 LDAP/PDF 功能），不阻断基本使用。
